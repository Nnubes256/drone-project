[−][src]Module icaros_control::web

Main webserver thread logic

Unlike the rest of the code, this thread runs on the Tokio asyncronomous event loop for better performance and semantics for the task.

To handle the web side itself, the warp web server library is used. Three endpoints are exposed from it:

  • /feed, which represents our core WebSocket endpoint, and streams the main system state and telemetry to connected users.
  • /video, which represents our video WebSocket endpoint, and streams the live video feed received from the drone through WifiBroadcast.
  • A catch-all endpoint, which serves the front-end side of the web application.
